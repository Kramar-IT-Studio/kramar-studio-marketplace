# /architect:c4

This command is an alias for `/architect:diagram c4-<level> <subject>`.

It generates exactly one C4 diagram. For other diagram types — sequence, state, ER, deployment — use `/architect:diagram <type> <subject>` directly.

## Usage

```
/architect:c4 context "the storefront and its surrounding services"
/architect:c4 container "the editor backend and its dependencies"
/architect:c4 component "the AI gateway internals"
```

These are equivalent to:

```
/architect:diagram c4-context "the storefront and its surrounding services"
/architect:diagram c4-container "the editor backend and its dependencies"
/architect:diagram c4-component "the AI gateway internals"
```

## Steps

1. Parse the first argument as the C4 level (`context`, `container`, or `component`).
2. Forward to `/architect:diagram` logic with type `c4-<level>`.

If the level argument is missing or invalid, ask the user to pick. Don't render multiple levels.

## Why both commands exist

`/architect:c4` predates `/architect:diagram`. It's kept as an alias so old workflows and muscle memory don't break. The substantive logic is in `/architect:diagram` and the `architectural-diagrams` skill.

For new work — use `/architect:diagram` directly. It's the single entry point for all five diagram types.
